系统概览

Mysdd 是一个家长端作业辅导应用。家长注册账号，给孩子建立档案，上传题目图片，后端调用多模态大模型（MiniMax-M3）生成解题思路，通过 SSE 实时流式推送给前端。

🎯 核心能力

  • 家长注册 / 登录 / JWT 鉴权
  • 孩子档案管理（CRUD）
  • 题目图片上传 + AI 求解
  • SSE 实时推理过程展示
  • 解题结果持久化 + 历史查阅

🏢 多租户模型

每个家长（User）拥有独立的孩子（Child）和题目（Problem）数据。通过 JWT 中的 userId 实现请求级租户隔离，IDOR 防御确保跨租户不可枚举。

安全设计

IDOR 防御 — 404 统一消息

flowchart LR A["请求: childId=7"] --> B{"findFirst({id, userId})"} B -->|"匹配"| C["✅ 通过"] B -->|"不匹配"| D["404 'child 不存在'"] E["请求: problemId=99"] --> F{"findFirst({id, child.userId})"} F -->|"匹配"| G["✅ 通过"] F -->|"不匹配"| H["404 'problem 不存在'"] style D fill:#f38ba833,stroke:#f38ba8 style H fill:#f38ba833,stroke:#f38ba8 style C fill:#a6e3a133,stroke:#a6e3a1 style G fill:#a6e3a133,stroke:#a6e3a1

🛡️ 安全原则

  • 统一 404：资源不存在 和 资源不属于你 → 同一响应（不区分），防止枚举攻击
  • 单查询 findFirst：WHERE id=? AND user_id=?，无 TOCTOU 窗口
  • 日志同形：server log 用 outcome: not_found_or_forbidden，不区分原因
  • 不回显 ID：错误消息不含 childId / problemId
  • SSE 端点前置 Guard：StreamIdorGuard 在 @Sse() 打开响应之前执行 IDOR 检查，避免 200 开流后才发现无权
  • 图片无公开 URL：不使用 app.useStaticAssets，所有图片访问经过认证 + IDOR

架构决策（ADR）

ADR-0004: 异步 + SSE 流式 AI 求解

Accepted 决策：采用异步 + SSE 流式方案。

选择对比

候选描述问题
A. 同步POST 等 AI 完成再返HTTP 60s 超时；阻塞 worker；关页无法取消
B. 异步+轮询POST 立返，2s 轮询 GET实时性差；无 thinking 可视化
C. 异步+SSEPOST 立返，订阅 stream✅ 选中 — 复杂度换体验

关键约束

  • POST /problems 只建行（status=pending），不预启动 AI
  • GET /problems/:id/stream 触发求解，返回 text/event-stream
  • 180s 硬超时（AbortController），不自叠 retry（信任 SDK 内置重试）
  • 并发抢占：updateMany 原子锁，count=0 → already_processing
  • 断线兜底：GET /problems/:id 返回当前 status + 终态 solution

ADR-0005: IDOR 防御采用 404 统一消息 + 单查询

Accepted 决策：统一 404 + 单查询 findFirst。

  • "不存在"和"不是你的" → 同一 404 + <资源> 不存在
  • 单条 findFirst 带复合 where，无 TOCTOU 窗口
  • 日志统一 outcome: not_found_or_forbidden，不区分
  • 反向条件：扩展到学校端（多 userId 共享资源）时引入 403

ADR-0006: Storage 层采用接口 + 本地磁盘实现

Accepted 决策：StorageService 接口 + LocalDiskStorageService 实现。

  • 业务代码注入 STORAGE_SERVICE token，不依赖具体类
  • 路径 <uuid> 而非 <problemId>，防 ID 泄露
  • 测试 fake: MemoryStorageDriver 不碰盘
  • 反向条件：用户 100w+ 时切 OSS/S3，只加实现 + 换 token
